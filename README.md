# 魔兽世界 3D 战术规划工具

> WoW-Style 3D Raid Tactics Planner

一款基于浏览器的魔兽世界风格 3D 团队副本战术规划工具。在 3D 场景中放置怪物和玩家单位、绘制标注（箭头/区域/标签）、管理多阶段战斗流程，并以动画演示方式展示完整战术。

---

## 功能亮点

- **3D 场景管理** — 加载 GLB/FBX 模型，支持纳克萨玛斯等副本地图，可上传自定义模型
- **单位放置系统** — 怪物（Boss、小怪、精英怪等）、玩家角色（坦克/治疗/输出及各职业）、自定义物品三大类
- **标注工具** — 方向箭头、圆形区域、文字标签，支持颜色和尺寸编辑
- **多阶段管理** — 每个场景可创建多个战斗阶段，阶段间切换带平滑过渡动画
- **一键演示** — 自动依次播放所有阶段，完整展示战斗时间线
- **自由漫游** — WASD 第一人称视角自由飞行，适合录制战术讲解视频
- **Z 轴剖切** — 裁剪模型上半部分，查看室内结构和单位站位
- **视角管理** — 保存/加载相机视角，支持分组管理和拖拽排序
- **场景导入导出** — 完整战术数据导出为 JSON，方便分享和备份

## 安装与运行

### 环境要求

- [Node.js](https://nodejs.org/) 18 或更高版本

### 安装

```bash
git clone <repo-url>
cd wow-3d
npm install
```

### 开发

```bash
npm run dev
```

浏览器打开 `http://localhost:5173`（Vite 默认端口）。

### 构建

```bash
npm run build    # 生成 dist/ 目录
npm run preview  # 本地预览构建结果
```

项目为纯前端应用，无需后端服务。

## 使用指南

### 场景管理

左侧边栏 **🗺️ 战斗场景** 面板管理场景：

- 项目预置了「团队副本」「大秘境」「PvP 战场」三个场景分组
- 点击场景卡片切换场景，自动加载对应的 3D 模型和阶段数据
- 拖拽 `.glb` / `.fbx` / `.gltf` 文件到上传区域可替换当前场景模型
- 拖拽多个模型文件可批量添加新场景
- 支持将场景导出为 JSON 文件，或从 JSON 导入场景数据

### 视图控制

左侧边栏 **👁️ 视图工具** 面板调整视角：

- **轨道控制**：鼠标左键旋转、滚轮缩放、右键平移
- **亮度调节**：滑块 0.3x ~ 3.0x
- **快捷视角**：一键切换俯视、正面、侧面、45° 等角视角

#### 自由漫游模式

点击 🚶 按钮或按 **F 键** 进入漫游模式：

| 按键 | 操作 |
|------|------|
| W / ↑ | 前进 |
| S / ↓ | 后退 |
| A / ← | 左移 |
| D / → | 右移 |
| Q | 下降 |
| E | 上升 |
| Shift | 加速 |

鼠标控制视角方向，再次按 F 退出。

#### Z 轴剖切

点击 ✂️ 按钮启停剖切面，拖动滑块在模型底部到顶部之间切割，查看建筑内部结构。

### 单位放置

左侧边栏 **🎯 单位放置** 面板放置战术单位：

1. 在「怪物」「玩家（角色/职业）」「自定义」标签页中选择单位类型
2. 点击单位卡片进入放置模式
3. 点击 3D 场景中的位置放置单位（自动吸附模型表面）
4. 按 **Esc** 退出放置模式

**单位交互：**
- 点击场景中的单位可选中（显示光环 + 脉冲动画）
- 拖动单位到新位置
- 选中后侧栏弹出变换面板：精确编辑坐标、旋转角度、缩放比例、标签名称
- 按 **Delete** 键删除选中单位

### 标注工具

左侧边栏 **✏️ 标注工具** 面板添加标注：

| 类型 | 操作方式 |
|------|---------|
| ➡️ 方向箭头 | 点击场景两次：起点 → 终点 |
| 🔴 区域标记 | 点击场景放置圆形区域，可调半径和颜色 |
| 📌 文字标签 | 点击场景放置文字标记 |

- 点击已放置的标注可编辑颜色或文字
- 按 **Delete** 删除选中标注

### 阶段管理

底部居中 **阶段条** 管理多阶段战斗：

- 点击阶段按钮切换阶段，自动保存当前状态并加载目标阶段
- 双击阶段名称可重命名
- 点击 **+ 阶段** 添加新阶段
- 点击 **▶ 演示** 自动循环播放所有阶段

### 视角管理

左侧边栏 **🎥 视角管理** 面板保存和恢复相机视角：

- 点击保存按钮记录当前相机位置
- 点击已保存的视角卡片跳转到对应视角
- 支持创建多个视角分组，组内视角可拖拽排序

### 键盘快捷键

| 快捷键 | 功能 |
|--------|------|
| F | 切换自由漫游模式 |
| W A S D | 漫游模式移动 |
| Q / E | 漫游模式下降/上升 |
| Shift | 漫游模式加速 |
| Delete / Backspace | 删除选中的单位或标注 |
| Escape | 取消放置模式 / 取消选中 |

## 技术架构

### 技术栈

- **构建工具**：[Vite 6](https://vitejs.dev/)
- **3D 渲染**：[Three.js 0.183](https://threejs.org/) (WebGL)
- **语言**：Vanilla JavaScript (ES Modules)
- **Draco 压缩**：GLTF 模型使用 Draco 解码器加载

无任何 UI 框架（React/Vue 等），无状态管理库，纯 DOM 构建 UI。

### 模块结构

```
src/
├── App.js                 # 入口：依赖注入组装所有模块
├── Constants.js           # 单位定义、常量数据（纯数据）
├── DataStore.js           # 场景/阶段/视角数据管理
├── SceneManager.js        # Three.js 渲染管线（相机、光照、动画）
├── ModelManager.js        # GLB/FBX 模型加载与上传
├── ClipPlaneManager.js    # Z 轴剖切面控制
├── UnitManager.js         # 单位 Chibi 模型创建、放置、拖拽
├── AnnotationManager.js   # 箭头/区域/标签标注
├── PhaseManager.js        # 阶段切换与动画过渡
├── ViewpointManager.js    # 相机视角保存与跳转
├── InteractionManager.js  # 射线检测与键鼠事件
└── UIManager.js           # 全部 DOM 界面构建
```

### 设计模式

- **依赖注入**：所有模块通过构造函数接收依赖，[App.js](src/App.js) 是唯一的组装点
- **单向数据流**：`DataStore` → `业务模块` → `渲染/UI 模块`
- **纯数据分离**：`Constants.js` 和 `DataStore.js` 不依赖 Three.js，可独立测试
